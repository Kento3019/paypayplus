# OKULINK 仕様書

> スプリント計画は [sprints.md](sprints.md) を参照。

---

## プロダクト概要

2人向けの支払い管理アプリ。「いくら払えばいい？」という会話と、PayPayでの宛先指定の手間をゼロにする。2人専用・ログインなし・URLを知っている人だけがアクセスできる。

---

## データ構造

```typescript
type Member = {
  id: string; // "m1" | "m2"
  name: string; // メンバー名（最大10文字）
  color: string; // カード縁の色（m1="#2196F3"青, m2="#F44336"赤）
};

type Room = {
  id: string; // URLハッシュ（roomId）
  createdAt: Date;
  members: [Member, Member]; // 2人固定
};

type Payment = {
  id: string; // UUID（crypto.randomUUID()）
  title: string; // タイトル（最大20文字）
  amount: number; // 金額（1以上999999以下）
  payPayUrl: string | null;
  createdAt: Date;
  completedAt: Date | null;
  isDone: boolean;
  creatorId: string | null; // "m1" | "m2" | null
};
```

## 画面・ルーティング

| 画面             | URL                   | 説明                           |
| ---------------- | --------------------- | ------------------------------ |
| ようこそ画面     | `/`                   | アプリ説明・ルーム作成         |
| ルーム作成画面   | `/#create`            | メンバー名2人登録・ルーム作成  |
| 共有リンク画面   | `/#share/{roomId}`    | URLコピー・ルームへの入室      |
| メンバー編集画面 | `/#settings/{roomId}` | メンバー名変更                 |
| ルームページ     | `/#abc123xyz`         | 支払い管理ダッシュボード       |
| 404ページ        | その他                | 不正ハッシュ・存在しないroomId |

- `/#share/{roomId}` / `/#settings/{roomId}`: roomId Firestore不在→404
- ルームIDは `generateRandomHash()`（`src/lib/routing.ts`）で生成
- FirebaseのAPIキーはソースに直書き
- 2人はURLを共有するだけ。ログイン不要

## ダッシュボード

未完了カードを作成日の新しい順に並べる。余計なテキストは一切なし。

```
┌║────────────────────┐  ← ║ = 4px縦ボーダー（作成者カラー）
│ ランチ               │
│ 5/1に作成            │  ← サブテキスト（小フォント）
│      ¥2,500          │  ← 中央・大きく
│    [PayPayで払う]    │
└─────────────────────┘
```

- カード左端4pxボーダー: m1=青`#2196F3`, m2=赤`#F44336`, null=グレー`#E0E0E0`
- 作成者名はcreatorId=nullのとき非表示
- PayPayリンクなし → ボタンをグレーアウト
- カード並び順: 作成日の新しい順

### 空状態

未完了0件のとき、新規カードフォームを自動展開。履歴のみ存在する場合も同様。
判定タイミング: Firestoreからpayments取得直後（毎回判定）。

### 履歴（アコーディオン）

- 完了済みが0件 → 履歴セクション非表示
- 完了日ベースで折りたたみ（月曜始まり）

| セクション       | デフォルト |
| ---------------- | ---------- |
| 今週（月曜〜）   | 開く       |
| 今月（今週除く） | 閉じる     |
| それ以前         | 閉じる     |

展開すると `ランチ ¥2,500 (5/1)` 形式で表示。

## カード操作

### 操作フロー

| 操作   | アクション                                                            |
| ------ | --------------------------------------------------------------------- |
| 追加   | FAB（＋）タップ → 編集状態カードがリスト先頭に出現 → 入力 → 保存      |
| 編集   | 右スワイプ → 青の編集ボタン → カードが編集状態に                      |
| 削除   | 右スワイプ → 赤の削除ボタン → 「削除しますか？」[はい][いいえ] → 削除 |
| 支払い | PayPayボタンタップ → PayPayへ遷移（完了は手動）                       |
| 完了   | 左スワイプ → 「支払い完了しましたか？」[はい][いいえ]                 |

編集中は他のカード操作・FABをロック。編集カード外タップでキャンセル。

### カード編集UI

```
┌─────────────────────┐
│ [ランチ          ]   │  ← 最大20文字
│ [¥ 2,500         ]   │  ← inputmode="numeric"
│ [paypay.ne.jp...][📥]│  ← ClipboardPaste（lucide, 20px）
│ [花子から太郎に支払う]│  ← creatorId = "m1"（太郎が立替）
│ [太郎から花子に支払う]│  ← creatorId = "m2"（花子が立替）
│ [キャンセル] [保存]  │
└─────────────────────┘
```

支払い方向ボタン仕様:

- 縦並び・全幅（`w-full`）
- 動的表示: `{m2.name}が{m1.name}に支払う` / `{m1.name}が{m2.name}に支払う`
- 選択中: 支払い元カラーで背景色付き（m1=青, m2=赤）
- `whitespace-normal`（折り返し許容）・`text-center`（名前10文字上限でも2行内に収まる）
- 未選択（null）も許容

### スワイプのビジュアルフィードバック

アイコンライブラリ: `lucide-react`

| スワイプ方向 | 背景色    | アイコン（lucide-react）   | アクション       |
| ------------ | --------- | -------------------------- | ---------------- |
| 左（完了）   | `#4CAF50` | `CheckCircle2`（白・32px） | 完了ポップアップ |
| 右（編集）   | `#2196F3` | `Pencil`（白・28px）       | 編集状態に       |
| 右（削除）   | `#F44336` | `Trash2`（白・28px）       | 削除ポップアップ |

- スワイプ距離0〜80pxでアイコン opacity 0→1
- FABアイコン: `Plus`（lucide-react）

## バリデーション

- **タイトル**: 必須・空文字不可・最大20文字・1行
- **金額**: `inputmode="numeric"`・数字のみ・1以上999,999以下
- **PayPayリンク**: `paypay.ne.jp` ドメインチェック・不正URLは赤枠＋エラー表示

| 場面           | アニメーション                                   |
| -------------- | ------------------------------------------------ |
| 完了・削除時   | 右へスライドアウト + フェードアウト（0.3s）      |

- 位置: 画面下部中央（fixed・safe-area-inset-bottom 考慮）
- デザイン: 角丸・`shadow-xl`・アイコン + テキスト横並び
- アニメーション: 下からスライドアップ → 2.5秒後に下へスライドアウト
- アイコン: `lucide-react`

| 操作       | メッセージ                   | アイコン             | 背景色    |
| ---------- | ---------------------------- | -------------------- | --------- |
| 保存       | 「保存しました」             | `Check`（緑）        | `#1B5E20` |
| 削除       | 「削除しました」             | `Trash2`（白）       | `#B71C1C` |

### モバイルUI制約

**縦スクロール**: カードコンテナに `touch-action: pan-y`・`preventScrollOnSwipe: false`（react-swipeable）。スワイプ判定は水平方向のみ。

**テキストオーバーフロー**:

- カードコンテナ: `overflow: hidden`・`min-width: 0`（flex子要素の縮小許容）
- タイトル: `line-clamp-2`（最大2行、超過は `…`）
- 金額・サブテキスト: `truncate`（1行）

## Firestoreデータ設計

```
/rooms/{hashId}                       ← Room ドキュメント
/rooms/{hashId}/payments/{paymentId}  ← Payment サブコレクション
```

Room ドキュメント:

```json
{
  "createdAt": "Timestamp",
  "members": [
    { "id": "m1", "name": "太郎", "color": "#2196F3" },
    { "id": "m2", "name": "花子", "color": "#F44336" }
  ]
}
```

- `paymentId`: UUID（`crypto.randomUUID()`）
- ハッシュはpaymentドキュメントには持たせない
- セキュリティルール: hashId一致のみ読み書き許可
- リアルタイム同期: `onSnapshot`
- ルームページ起動時にRoomドキュメントを読み込み、不在→404
