# PayPay支払い管理アプリ 仕様書

---

## 1. プロダクト概要

2人向けの支払い管理アプリ。「いくら払えばいい？」という会話と、PayPayでの宛先指定の手間をゼロにする。2人専用・ログインなし・URLを知っている人だけがアクセスできる。

---

## 2. データ構造

```typescript
type Member = {
  id: string; // "m1" | "m2"
  name: string; // メンバー名（最大10文字）
  color: string; // カード縁の色（m1="#2196F3"青, m2="#F44336"赤）
};

type Room = {
  id: string; // URLハッシュ（roomId）
  createdAt: Date; // 作成日
  members: [Member, Member]; // 2人固定
};

type Payment = {
  id: string; // UUID（crypto.randomUUID()）
  title: string; // タイトル（最大20文字）
  amount: number; // 金額（1以上999999以下）
  payPayUrl: string | null; // PayPayリンク（optional）
  createdAt: Date; // 作成日
  completedAt: Date | null; // 完了日
  isDone: boolean; // 完了フラグ
  creatorId: string | null; // "m1" | "m2" | null
};
```

## 3. 画面構成

### 画面一覧

| 画面           | URL                 | 説明                             |
| -------------- | ------------------- | -------------------------------- |
| ようこそ画面   | `/`（ハッシュなし） | アプリ説明・ルーム作成           |
| ルーム作成画面 | `/#create`          | メンバー名2人登録・ルーム作成    |
| 共有リンク画面   | `/#share/{roomId}`    | URLコピー・ルームへの入室        |
| メンバー編集画面 | `/#settings/{roomId}` | メンバー名変更                   |
| ルームページ     | `/#abc123xyz`         | 支払い管理ダッシュボード（既存） |
| 404ページ        | その他                | 不正ハッシュ                     |

### ダッシュボード

未完了カードを作成日の新しい順に並べる。余計なテキストは一切なし。

```
┌─────────────────────┐
│ ランチ              │
│ 花子さんが5/1に作成   │←タイトルより小さいフォント。メンバーと作成日(更新日)を表示。
│      ¥2,500          │←金額は中央に大きく
│                      │
│    [PayPayで払う]    │
└─────────────────────┘
```

### 空状態

未完了カードが0件のとき、新規カードフォームを自動展開（`## 22`）。履歴のみ存在する場合も同様。

### 履歴（アコーディオン）

- 完了済みのみ対象
- 完了日ベースで折りたたみ（月曜始まり）
- 完了済み0件の場合は履歴セクション自体を非表示

| セクション       | デフォルト |
| ---------------- | ---------- |
| 今週（月曜〜）   | 開く       |
| 今月（今週除く） | 閉じる     |
| それ以前         | 閉じる     |

展開すると `ランチ ¥2,500 (5/1)` のみ表示。

## 4. カード編集状態

追加・編集時、カードがその場で編集可能状態になる。新規追加はリスト先頭に出現。編集中は他のカード操作・FABをロック。編集カード外タップでキャンセル扱い。

```
┌─────────────────────┐
│ [ランチ          ]   │  ← 最大20文字
│                      │
│ [¥ 2,500              ] │  ← inputmode="numeric"
│                      │
│ [paypay.ne.jp...][📥]│  ← ペーストボタン（lucide: `ClipboardPaste` 20px）
│                      │
│ [キャンセル] [保存]  │
└─────────────────────┘
```

---

## 5. 操作フロー

| 操作       | アクション                                                                               |
| ---------- | ---------------------------------------------------------------------------------------- |
| **追加**   | FAB（＋）タップ → 編集状態のカードがリスト先頭に新規出現 → 入力 → 保存                   |
| **編集**   | 右スワイプ → 青の編集ボタン → カードが編集状態に                                         |
| **削除**   | 右スワイプ → 赤の削除ボタン → 確認ポップアップ `「削除しますか？」[はい][いいえ]` → 削除 |
| **支払い** | PayPayボタンタップ → PayPayへ遷移（完了は手動）                                          |
| **完了**   | 左スワイプ → 確認ポップアップ `「支払い完了しましたか？」[はい][いいえ]`                 |

## 6. スワイプのビジュアルフィードバック

スワイプ中に背景色とアイコンが出現し、何が起きるかを直感的に示す。

- アイコンライブラリ: `lucide-react`

| スワイプ方向 | 背景色       | アイコン（lucide-react）         | アクション       |
| ------------ | ------------ | -------------------------------- | ---------------- |
| 左（完了）   | 緑 `#4CAF50` | `CheckCircle2`（白・32px）       | 完了ポップアップ |
| 右（編集）   | 青 `#2196F3` | `Pencil`（白・28px）             | 編集状態に       |
| 右（削除）   | 赤 `#F44336` | `Trash2`（白・28px）             | 削除ポップアップ |

- スワイプ距離に応じてアイコン opacity 0→1（距離0〜80px でフェードイン）
- 削除 FAB（＋）も `Plus` アイコン（lucide-react）使用

ライブラリ: `framer-motion`（`AnimatePresence` + `motion.div` を使用）

## 12. 部屋・認証

| URL                   | 遷移先         |
| --------------------- | -------------- |
| `/`（ハッシュなし）      | ようこそ画面     |
| `/#create`               | ルーム作成画面   |
| `/#share/{roomId}`       | 共有リンク画面（roomId Firestore不在→404）|
| `/#settings/{roomId}`    | メンバー編集画面（roomId Firestore不在→404）|
| `/#abc123xyz`（有効）    | ルームページ     |
| その他                   | 404ページ        |

- ルーム作成時に既存の `generateRandomHash()`（`src/lib/routing.ts`）でroomIdを生成
- FirebaseのAPIキーはソースに直書き
- 2人はURLを共有するだけ。ログイン不要
- Firestoreに存在しないroomIdへアクセスした場合も404ページを表示

## 14. Firestoreデータ設計

```
/rooms/{hashId}                       ← Room ドキュメント
/rooms/{hashId}/payments/{paymentId}  ← Payment サブコレクション
```

**Room ドキュメント構造**

```json
{
  "createdAt": "Timestamp",
  "members": [
    { "id": "m1", "name": "太郎", "color": "#2196F3" },
    { "id": "m2", "name": "花子", "color": "#F44336" }
  ]
}
```

- `paymentId` はUUID（`crypto.randomUUID()`）
- ハッシュはpaymentドキュメントには持たせない
- FirestoreのセキュリティルールでhashId一致のみ読み書き許可
- リアルタイム同期は `onSnapshot` で実装
- ルームページ初期化時にRoomドキュメントを読み込み、存在しなければ404へ遷移
