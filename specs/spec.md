# PayPay支払い管理アプリ 仕様書

---

## 1. プロダクト概要

カップル専用の支払い管理アプリ。「いくら払えばいい？」という会話と、PayPayでの宛先指定の手間をゼロにする。2人専用・ログインなし・URLを知っている人だけがアクセスできる。

---

## 2. データ構造

```typescript
type Payment = {
  id: string; // UUID（crypto.randomUUID()）
  title: string; // タイトル（最大20文字）
  amount: number; // 金額（1以上999999以下）
  payPayUrl: string | null; // PayPayリンク（optional）
  createdAt: Date; // 作成日
  completedAt: Date | null; // 完了日
  isDone: boolean; // 完了フラグ
};
```

---

## 3. 画面構成

### ダッシュボード

未完了カードを作成日の新しい順に並べる。余計なテキストは一切なし。

```
┌─────────────────────┐
│ ランチ              │
│ 5/1に作成            │←タイトルより小さいフォントサイズ。作成日(更新日)を表示。
│      ¥2,500          │←金額は中央に大きく
│                      │
│    [PayPayで払う]    │
└─────────────────────┘
```

### 空状態

カードが0件の時、常に中央に小さく表示。

```
URLを共有してね
```

### 履歴（アコーディオン）

- 完了済みのみ対象
- 完了日ベースで折りたたみ（月曜始まり）
- 完了済み0件の場合は履歴セクション自体を非表示

| セクション       | デフォルト |
| ---------------- | ---------- |
| 今週（月曜〜）   | 開く       |
| 今月（今週除く） | 閉じる     |
| それ以前         | 閉じる     |

展開すると `ランチ ¥2,500 (5/1)` のみ表示。

---

## 4. カード編集状態

追加・編集時、カードがその場で編集可能状態になる。新規追加はリスト先頭に出現。編集中は他のカード操作・FABをロック。編集カード外タップでキャンセル扱い。

```
┌─────────────────────┐
│ [ランチ          ]   │  ← 最大20文字
│                      │
│ [¥ 2,500        ]   │  ← inputmode="numeric"
│                      │
│ [paypay.ne.jp...][📋]│  ← ペーストボタン
│                      │
│ [キャンセル] [保存]  │
└─────────────────────┘
```

---

## 5. 操作フロー

| 操作       | アクション                                                                               |
| ---------- | ---------------------------------------------------------------------------------------- |
| **追加**   | FAB（＋）タップ → 編集状態のカードがリスト先頭に新規出現 → 入力 → 保存                   |
| **編集**   | 右スワイプ → 青の編集ボタン → カードが編集状態に                                         |
| **削除**   | 右スワイプ → 赤の削除ボタン → 確認ポップアップ `「削除しますか？」[はい][いいえ]` → 削除 |
| **支払い** | PayPayボタンタップ → PayPayへ遷移（完了は手動）                                          |
| **完了**   | 左スワイプ → 確認ポップアップ `「支払い完了しましたか？」[はい][いいえ]`                 |

---

## 6. スワイプのビジュアルフィードバック

スワイプ中に背景色とアイコンが出現し、何が起きるかを直感的に示す。

| スワイプ方向 | 背景色       | アイコン | アクション       |
| ------------ | ------------ | -------- | ---------------- |
| 左（完了）   | 緑 `#4CAF50` | ✓        | 完了ポップアップ |
| 右（編集）   | 青           | ✏️       | 編集状態に       |
| 右（削除）   | 赤           | 🗑️       | 削除ポップアップ |

---

## 7. アニメーション

| 場面   | アニメーション                       |
| ------ | ------------------------------------ |
| 完了時 | カードがフェードアウトして履歴へ移動 |
| 削除時 | カードがフェードアウトして消える     |

---

## 8. トースト通知

| 操作                   | メッセージ                   |
| ---------------------- | ---------------------------- |
| 保存                   | 「保存しました」             |
| 削除                   | 「削除しました」             |
| 完了                   | 「完了しました」             |
| オフライン・通信エラー | 「通信エラーが発生しました」 |

---

## 9. バリデーション

- **タイトル**：必須・空文字は弾く・最大20文字・1行切り
- **金額**：`inputmode="numeric"`・数字のみ・1以上999,999以下
- **PayPayリンク**：`paypay.ne.jp` ドメインチェック・不正URLは赤枠＋エラー表示

## 11. 金額フォーマット

`¥2,500`（3桁カンマ区切り・円マーク付き）

---

## 12. 部屋・認証

```
/#abc123xyz  → 固定の本番部屋（ソースに直書き）
/#           → ランダムハッシュを自動生成してリダイレクト（デモ用）
```

- ハッシュなし・不正ハッシュは404ページ
- FirebaseのAPIキーもソースに直書き
- 2人はURLを共有するだけ。ログイン不要

## 14. Firestoreデータ設計

```
/rooms/{hashId}/payments/{paymentId}
```

- `paymentId` はUUID（`crypto.randomUUID()`）
- ハッシュはpaymentドキュメントには持たせない
- FirestoreのセキュリティルールでhashId一致のみ読み書き許可
- デモ用自動生成部屋も同じルールで許可
- 同時編集の競合は許容（個人利用のため）
- リアルタイム同期は `onSnapshot` で実装

---

| 項目             | 挙動                       |
| ---------------- | -------------------------- |

| オフライン時     | トーストでエラー表示       |
| カード並び順     | 作成日の新しい順           |
